# 规则配置说明

## 基础奖励（档位 / CPM / 瓜分）
- 档位：按渠道/平台匹配阈值金额。字段示例：`阈值`、`奖励`。
- CPM：按播放量/千 * CPM，渠道可配置不同 CPM，示例字段 `抖音/视频号`、`小红书`、`B站`。
- 瓜分：达到门槛的作品按播放量占比分配奖金池。参数：`total`（池总额）、`min_play`（门槛）。

## 优秀奖励
- 字段：`阈值`、`加成`、`仅短视频`（布尔）。满足阈值则加成，可限制短视频适用。

## 限时奖励
- 字段：`播放下限`、`加成`。达到播放下限时叠加奖励。

## 计算顺序（摘要）
1. 基础奖励：按选定模式计算（档位/CPM/瓜分）。
2. 优秀奖励：满足条件叠加。
3. 限时奖励：满足条件叠加。

## 数据要求（上传文件）
- 必需列：渠道/平台、播放量、作品类型，且账号ID/账号名称/账号昵称 至少一列。
- 可选列：点赞、评论数、期数、视频标题/作品标题、B站热搜/热门。

## UI 编辑入口
- 基础奖励：Tabs 中的 “基础奖励” 选择模式并填写参数/表格。
- 优秀奖励：Tab “优秀奖励” 中编辑表格（支持动态行）。
- 限时奖励：Tab “限时奖励” 中编辑表格。

## 导出
- 计算后可下载处理过的 Excel（带奖励结果与字段）。
